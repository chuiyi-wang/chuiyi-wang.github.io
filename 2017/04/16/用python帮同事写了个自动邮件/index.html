一点点感想

先做到“正确”

写这个小程序，起因是同事请我帮忙把他的周报写成程序，以免去每周四的重复劳动之苦。这个需求，乍一看就是个透视表，没什么难度。但是要做到全自动邮件，就必须考虑很多潜在情况，这里着实花了一些时间。

几个小细节：

  • 数据清洗时，去除了因延迟导致的“有点击无曝光”的数据。
  • 透视表里的数，根据业务需求，基本上是CTR，有一列是VTR。
  • 透视表最后一列的均值，不是本列所有行的点击率之平均，而是该广告类型下的sum(click or trueview)/ sum(曝光)
  • 若任何一个点击率的曝光量低于100，那么点击率很可能存在偏差，会置灰提醒。

再做到“易用”

我自己的周报虽然早已脚本化了，但是写得简陋，基本不考虑用户体验，反正用户是我自己。。。而给别人写的程序，就会比较仔细考虑每个步骤下的人机交互，也要避免因为没有考虑到可能的报错情况，而带来用户恐慌情绪。。。

一个小细节：

  • 所谓程序初始化，目的就是为了让同事每周都回答一次大锤“棒棒哒！”，以达到洗脑功效23333333

最后还要“能看”

一开始发出来的邮件，是这样的：

太棒了，缺点就是界面复古了点……

但我毕竟是个有点审美的产品经理，所以该改还是得改改。结果这一改，一天就过去了……

目前是用pandas自带的to_html来生成表的html。想起我第一次写自动邮件，html竟然是强行写的，心疼蠢蠢的自己。

样式植入则采用内部样式表（位于\标签内部）和内敛样式（在HTML元素内部）两种方式，也是够用了。

其实每次做前端的工作，我都会思考：程序员的“直男审美”，到底是天生的，还是因为懒得搞前端而丧失基本审美的呢。。。😂
