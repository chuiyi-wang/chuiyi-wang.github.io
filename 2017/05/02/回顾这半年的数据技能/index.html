回顾这半年的数据技能

发表于 2017-05-02

过去半年，我花20%的精力搞数，30%的精力提高搞数能力，40%的精力被业务搞，而用于提升广告专业知识的精力，不超过10%。

那30%的精力还是有些成效，我以“精通excel”的能力为起点，基于工作需要，拓展出了以下能力：

1.MySQL的基本查询

对MySQL，不敢说“精通”，毕竟仍然经常被BI同事打开新技能的大门。但是作为多个团队的数据支持者，我还是称职的。

2.Python

Python目前在我心中是一个扛把子的地位。对于Python的学习，除了一些常用语法、库，主要掌握了以下几块知识点：

  • 爬虫

    用爬虫实现过：科大教务系统成绩统计、自如找房、为“成语接龙程序”上网爬素材、公司内网下爬周报数据。

  • pandas做数据处理

    学pandas的起因是好友黄老板的一句话，她说要是我能用pandas做数据处理，就可以秒杀99%的产品经理。爱装逼如我，怎么会放弃这个机会呢！于是我就去尝试了，结果发现在数据处理方面，它确实比MySQL顺手一些。现在用pandas也是比较熟练了，能满足日常的各种搞数需求。

  • 正则表达式

    正则表达式实在好用，不仅爬虫需要，日常的数据处理也是经常用到。例如我在今年2月做了份机型定向的数据调研。由于BI对于安卓机型名称的记录，完全是取值于手机自带的名称，因此要在三万多个杂乱的字符串中，整理出当前各个安卓品牌在奇效广告平台上的消耗，人工整理肯定要累死。但是有了正则表达式，就可以把每个品牌的机型名称总结成几个规律，如：

    一两天搞定，妥妥的。

  • 自动邮件

    主要是用到了smtplib库，和一点点基本的html、css知识。自动邮件的学习不是最难的，但性价比确实最高的。平均每周能省下五六个小时。

虽然自知水平有限，但我对技术的学习一向充满热情。对我来说，得到以上的能力，就仿佛步行的人学会开车，力量得到了惊人的放大。有段时间我经常深夜在公司敲代码，以至于引起BI同事的注意和转岗邀约（当然是玩笑）。

入行以来，我对自己的期望，一直是成为一个在广告行业内有影响力的PM。做数据处理，是leader为我设计的道路起点，我也颇为喜欢。现在对于搞数的技能点，我也点了一些，基本能涵盖日常的数据处理需求了，自认为（坐井观天式）数据处理的能力确实能超过80%的产品经理。而现在，我要给自己有新的期望，那便是——对计算广告的知识掌握，有全新的成长速度。
