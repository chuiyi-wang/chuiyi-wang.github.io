只要一小时，谁都能轻松建立个人网站

发表于 2016-03-20   |   分类于 Tracy工具箱

作为一个非计算机专业的学生，第一次在知乎上知道wordpress这个神器的时候简直惊喜，当即照着前辈们的教程着手建立我的个人网站（http://chuiyi.wang）。但是一路跟下来，我发现这些教程并不足够满足我的需求：便宜、简单、快捷。所以特开此帖，给像我一样想要快速建立个人网站的非计算机系学生党。目前为止，除了注册域名必须花的18元，尚无其他支出。

一般的教程主要步骤如下：

  1. 注册域名

  2. 注册虚拟空间

  3. 下载并安装wordpress

  4. 建立数据库

  5. 解析域名

  6. 安装模板和插件

今天我要告诉大家另一样神器：新浪云。它可以帮助我们轻松搞定2、3、4个步骤。

所以这个教程的主要步骤如下：

  1. 注册域名

  2. 注册和使用新浪云

  3. 解析域名

  4. 安装模板和插件

1. 注册域名

国内外的域名注册商都是可以选择的，我个人使用的是：

阿里云域名注册

注册方法很简单，搜索并筛选未被注册的域名，然后付款就行。一个普通的域名的价格一般是第一年40、50元，也有的域名比较便宜，像我注册的就是第一年18元。

2. 新浪云

在使用新浪云之前，为了注册虚拟空间，我在阿里做活动时申请了一年的免费虚拟空间，申请国内的主机空间比较麻烦的一点，就是要备案。备案过程特费时，一来一回了好几天，在此按下不表。新浪云好的一点就是目前对个人开发者并没有备案要求。

关于虚拟空间，另一个要考虑的因素是，一年后阿里云就会开始收费，不管你的PV（page view）是zero还是爆表，都要固定缴费。而新浪云使用的是“按需收豆”的收费方式。1元 = 100云豆，云豆可以用来购买新浪云的各种服务，包含新浪云应用 SAE、云主机 SEC、云存储 SCS、云容器 SC2、云商店 SCM、云邮箱 SEM。对于一个个人网站上来说，一两百颗云豆妥妥够用了。

为了建立数据库，我曾经使用过XAMPP，一个挺大的软件。但实际上如果你没有其他用途的话，下载这样一个软件就是杀鸡偏用牛刀了。而新浪云可以让我们不用操心数据库的问题。

那么现在开始使用新浪云创建wordpress。

首先，你需要注册新浪云：

点击“创建新应用”。

输入一个二级域名。如果你有自己的域名，这里可以随便填，以后我们会把网站改成独立域名。

填入应用名称，选择“WordPress for SAE"

填好要求信息提交之后，wordpress会一脸贱兮兮地告诉你：是不是还沉浸在安装的喜悦当中呀？可是你已经安装好了哦！

现在你返回控制台，就可以看到刚刚创建好的wordpress应用了。（我由于个人手贱，先后创建了两个，在下个步骤中会解释怎么手贱的）。

点击绑定。之后系统会弹出这个提示框：

这个信息不用特地记，我们返回域名管理还会看到：

那么怎么进行CNAME，怎么进行A解析呢？

这时候我们就需要回到我们注册域名的地方。我以阿里云域名注册为例。

首先，进入阿里云的管理控制台，找到你的域名：

点击域名，找到它的解析设置：

点击“添加解析”，即上图中的红色按钮。按照新浪云给的方式添加那两条解析。

等上几分钟，等到“是否生效”和“是否验证”都显示了“是”，就算解析完成了。这时在浏览器中输入你的独立域名，就可以链接到网站了。

update: 根据几位读者的反馈，在这里补充几条tips。

  1. 新浪云提供的CNAME和A解析地址由英文和数字组成（一定不是乱码），如果出现乱码请换个浏览器试试。

  2. CNAME主机记录那里填"chuiyi.wang"（你的独立域名）或"@"的意思都是一样的。

  3. 新浪云提供的A解析地址要复制全。有朋友看到我的阿里云A解析记录里的“主机记录”没有后面那串“.chuiyi.wang”，就跟着只复制前一半了。其实我是复制全了的，只是阿里云没有显示。

  4. 如果你发现虽然新浪云显示了“步骤进行到 是否验证：是 是否生效： 是”，但却无法通过独立域名打开网站。请检查一下阿里云解析记录里是否还有其他解析内容，有的话就删掉，只留新浪云要求解析的那两条，然后等几分钟看看。

但还有个问题，就是虽然输入个人域名可以进入到个人网站了，可是在网站里跳转一下，浏览器上方又变成了新浪给的二级域名。

这个问题很好办。回到个人网站的仪表盘，点击设置：

将URL填写为你的个人域名，保存后重新登录即可。

特别提醒好奇宝宝们：

千万不要填入其他网站的URL！

否则就再也找不到你自己的网站了……

对，这就是为什么我又重新建了第二个wordpress应用……

4. 安装模板和插件

到第三步为止，如果你对系统自带的网站主题可以接受，那个人建站就算完成了。如果你对网站主题有更高的追求，请往下看。

新浪云建站的缺陷是，无法在线选择wordpress主题和插件（其实在线模板种类也有限），所以如果想安装样式丰富的模板，我们需要另一样神器：SVN。

之前用阿里云虚拟空间的时候，不论是安装wordpress还是上传开源模板，方法都是ftp。ftp在windows上的操作比较简单，用本地文件夹就可以操作。OS系统本身对ftp没有支持，所以我用了一款广受推荐的叫FileZilla的免费软件。文件的上传速度极慢，忍无可忍的我又去尝试了其他ftp免费软件，发现都一样龟速。

所以现在初尝甜头的我一定要向各位安利新浪云+SVN的方式！

SVN一般来说是开发团队用以进行版本控制的工具。而在这个案例中，我们主要使用SVN来控制云端代码。

首先，我们需要下载一个SVN客户端。

Windows的用户请参考这一篇SAE的代码部署手册：（直接拖到“使用SVN客户端”）

SAE的代码部署手册

下载SmartSVN

SmartSVN一开始有30天的专业版试用期，到期会自动退为普通版。没关系，够用的。

说到SVN，几个名词稍微解释一下。

Repository（源代码库）－源代码统一存放的地方；

Checkout （提取）－当你手上没有源代码的时候，你需要从repository checkout一份；

Directories - 本地存储代码的地方

Commit （提交）－当你已经修改了代码，你就需要Commit到repository；

Update (更新)－当你已经Checkout了一份源代码，

Update一下你就可以和Repository上的源代码同步，你手上的代码就会有最新的变更。

简单来说，我们现在需要从新浪云（Repository）提取（Checkout）网站的源代码，放到本地（Directories），然后在本地进行代码修改（或者添加主题、插件文件），再将本地的文件提交（Commit）到新浪云。

所以打开SmartSVN后，我们就跟着软件引导进行Checkout.

这个地址可以在新浪云应用管理中的代码管理里找到：

然后输入本地存放代码的位置：

接着一路Continue……你会看到左下角正在Checkout：

完成之后，云端的文件就会出现在你的本地文件夹里了。

在本地文件夹里，可以找到wp-content这个文件夹，点开之后里面有叫做theme和plugins的文件夹，分别用来存放wordpress的主题和插件。接下来我们需要做的，就是在网络上下载心仪的主题和插件，然后放到theme或plugins文件夹里。

放好之后，需要把本地的代码传到云端。只需要打开SmartSVN，在左上角的Directories中找到你刚刚放进去的主题文件夹，进行Commit操作即可。

Commit成功后，回到wordpress仪表盘的“外观”管理中，你会发现可用主题除了最初默认那2个外，还加入了你刚放进去的主题。

用这个方法，你就可以玩遍所有开源的wordpress主题和插件了，想什么风格是什么风格~
