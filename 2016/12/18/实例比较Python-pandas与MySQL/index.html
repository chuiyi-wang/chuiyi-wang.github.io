比较Python Pandas与MySQL

发表于 2016-12-18   |   分类于 学代码的Tracy

其实对Python pandas也只是初玩，功能还没有探索全面，基本上只是把原来MySQL做的case，用Python pandas又重新写了一遍。因此对两种数据处理的方式有了初步的感官。

这些例子还是简单了点，事实上，越是复杂的需求，Python的简洁性就越会凸显。

不过在可读性上，我个人是认为MySQL更好读的。用pandas写出来的代码，括号和中括号经常混合出现，看得人眼花缭乱。

更灵活的取数方式

举个例子：对一个数据矩阵进行取数，MySQL竖着取是这样的：
SELECT columns FROM table，横着取是这样的：SELECT * FROM table LIMIT 5

对此，pandas憨憨一笑，给出了n个维度的取数方式。你可以切片取数，可以按列名，也可以按位置。简单来说，想象一下你任意拉个矩形压在表上，被压上的数据一定能被pandas轻松读出来。

很多时候，虽然原始表里有几十列，但我只需要每张表里的特定两三列数据。用MySQL跑出来的，虽然只SELECT了特定的那几列，但实际上使用的是INNER JOIN后的一张巨表，很是吃力。而在Python中，可以在一开始连接数据库时，便只读取想要的那几列数据。然后拿着df1, df2, df3…这几张小表自己玩。让我惊喜的是，原来在MySQL加了句NOT EXISTS之后就不堪负荷、再无音信的数据需求，用pandas却能直接就出结果。

分析小达猫

MySQL无疑在数据的存储和读取上有天然优势，但是读到数之后，分析的工作最好交给pandas。

Python作为一门脚本语言，自带的功能自然是丰富于MySQL的。比如更全面的operation函数，引以为傲的正则表达式等等。

此外，Python良好的交互性也是数据分析的优秀助攻。许多情况下，做分析并不能直接知道结果，所以就存在“试”的过程。例如当我用IPython跑数时，可以中途突然试一下：

根据Out出的一个超出预期的数字，我就知道中间一定有蹊跷，排查范围大概在哪里。然而用MySQL的话，就会像一个黑箱子实验，只知输入和输出，难以判断中间哪里出了问题，耗费大好时光。

Python连接一切

原来用MySQL处理数据，想联立两张在不同数据库的表，唯一的方法是导出其中一份表，放到另一个数据库。用Python就不需要这么麻烦，愿意导入什么都可以。

不仅在输入端方便，输出方面也是amazing极了。我经常面对这样的数据需求：除了得到统计结论之外，还要把各个意义下的明细数据列出。用MySQL时，我的方法是一个维度一个维度地跑，然后不厌其烦地对每个维度的query result进行export。用Python呢，操作变得异常简单：写好代码然后运行，立刻几份csv文件就会乖乖地自动出现，躺在文件夹里等候查阅。省出来的大片时间，真让人无所适从呢~


以上大概就是Python pandas带给我的惊喜。接下来我的学习方向是splinter，模拟浏览器行为，往连接一切的方向上再迈一步！
