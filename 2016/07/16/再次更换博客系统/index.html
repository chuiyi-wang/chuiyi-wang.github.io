再次更换博客系统 -- hexo

发表于 2016-07-16

我是被wordpress带进自建博客的世界的，不过接触了一段时间之后，我很快就改用了Typecho。据说全世界有三分之一的网站都是以wordpress为基础开发的，这也导致了它的代码过于万能，也就是厚重。相对于wordpress，Typecho的优势正如其官网所言：

然而好景不长，最近Typecho也被我抛弃了。这是因为在服务器的选择上，没有好的方案。一开始因为对流量的不确定，我选择了SAE的云豆扣费方式，为此还写了一篇Wordpress建站教程，植入SAE邀请链接，陆续得到了小几千颗云豆。然而网站在未进行SEO的情况下，扣豆方式莫名其妙。靠好友邀请得到的云豆支持了三个月之后，实在难以为继，所以决定转战hexo。

Hexo利用的是有300M免费静态空间的Github Page的，使用Markdown语法，图片存储在七牛上解决，国内外都可以较快访问。尤其对于个人博主来说，完全没有任何费用，从此远离“云豆耗尽报警”！

一开始之所以没有选择hexo，是因为对Node.js完全不了解，觉得不像wordpress和Typecho那样看着不顺眼了可以去查查代码。不过我发现自己在使用typecho的时候也没有进行过什么修改，所以这算个伪需求。

这次在安装hexo的过程中，一开始困惑很多也很蠢，比如我直接把
$ npm install -g hexo-cli连’$’一起拷进terminal里面，然后去Segmentfault里面查了N久-bash: $: command not found
的解决方式……再比如下好了npm之后，在文件夹里找了半天npm怎么打开……

不过最后搞明白了之后，现在超级喜欢terminal！比如原本代码部署需要打开svn才能commit，现在只需要一行$ hexo d即轻松搞定!!!

新博客的安装进了很多坑。已有教程很多，我就不写了，在这分享几个比较靠谱的教程吧：

  • hexo官网
  • 小白独立搭建博客–Github Pages和Hexo简明教程
  • 如何搭建一个独立博客——简明Github Pages与Hexo教程
